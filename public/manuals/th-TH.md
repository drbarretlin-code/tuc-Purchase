# ข้อมูลจำเพาะการจัดซื้อ TUC และระบบฐานความรู้ - คู่มือการใช้งาน

ยินดีต้อนรับสู่ระบบจัดซื้ออัจฉริยะของ TUC คู่มือนี้จะช่วยให้คุณเชี่ยวชาญฟังก์ชันหลักของระบบ เพื่อสร้างข้อกำหนดที่มีประสิทธิภาพและเป็นมาตรฐาน

---

## 1. การตั้งค่า Gemini API Key (ขั้นตอนสำคัญที่สุด)
ระบบนี้ใช้โมเดล **BYOK (Bring Your Own Key)** เพื่อให้มั่นใจในการประมวลผลข้อมูลที่มีประสิทธิภาพและความเป็นส่วนตัว

### ทำไมต้องใช้ Free Tier (ระดับฟรี)?
- **ไม่มีค่าใช้จ่าย**: สมัครผ่าน Google AI Studio ได้ฟรี เหมาะสำหรับการใช้งานประจำวัน
- **ประสิทธิภาพสูง**: มีความสามารถในการแยกวิเคราะห์ข้อกำหนดทางเทคนิคและการแปลหลายภาษาที่ทรงพลัง
- **ความสำคัญ**: หากไม่มี Key คุณจะไม่สามารถใช้ฟังก์ชันการสร้าง AI, การทำเหมืองข้อมูลขั้นสุด และการแปลแบบไดนามิกได้

### วิธีสมัครและตั้งค่า
1. ไปที่ **[Google AI Studio](https://aistudio.google.com/app/apikey)**
2. หลังจากเข้าสู่ระบบ ให้คลิก **"Create API key"** และคัดลอกไว้
3. คลิกไอคอน **(Settings ⚙️)** ที่มุมขวาบนของระบบ วางคีย์ลงในช่องและบันทึก

---

## 2. การจัดการไฟล์และการอัปโหลด
ในส่วน **(File Options 📁)** ที่ด้านบนของส่วนต่อประสานหลัก คุณสามารถดำเนินการดังต่อไปนี้:

- **(Cloud ☁️) โหลดจากคลาวด์**: ค้นหาไฟล์ประวัติในฐานข้อมูลและนำเข้าสู่พื้นที่แก้ไขหลักโดยตรงเพื่อแก้ไขต่อ
- **(Import 📂) โหลด JSON ในเครื่อง**: นำเข้าแบบร่างข้อกำหนดที่บันทึกไว้ก่อนหน้านี้เพื่อแก้ไขต่อ
- **(Export 💾) ส่งออกไฟล์**: ส่งออกข้อกำหนดที่เสร็จสมบูรณ์เป็นรูปแบบ JSON เพื่อการเก็บถาวรและการแลกเปลี่ยน

---

## 3. พื้นที่แก้ไขหลัก: เคล็ดลับการดึงข้อมูล AI และการอ้างอิง
เมื่อกรอกข้อกำหนดทางเทคนิค คุณสามารถใช้ **แผงช่วยเหลือ AI** ทางด้านขวาได้:

- **การจับคู่ข้อมูลประวัติ (History Data)**: ระบบจะเรียกคืนบันทึกการบำรุงรักษาอุปกรณ์ที่คล้ายกันในอดีตจากคลาวด์โดยอัตโนมัติ ตาม "ชื่ออุปกรณ์" ที่คุณป้อน
- **การอ้างอิงมาตรฐานทางเทคนิค (Technical Standards)**: สำหรับส่วนประกอบหรืองานวิศวกรรมเฉพาะ AI จะแนะนำมาตรฐาน KCG หรือบรรทัดฐานอุตสาหกรรมที่เกี่ยวข้อง
- **เคล็ดลับการใช้งาน**:
    - ปรับ **(Threshold ⚡)**: เพิ่มเปอร์เซ็นต์เพื่อการจับคู่ที่แม่นยำยิ่งขึ้น ลดเปอร์เซ็นต์ลงเพื่อการอ้างอิงที่กว้างขึ้น
    - คลิกปุ่ม **(Apply ➕)** ข้างรายการที่แนะนำเพื่อกรอกข้อมูลลงในฟิลด์ที่เกี่ยวข้องโดยตรง ช่วยลดเวลาในการพิมพ์ได้อย่างมาก

## 4. พื้นที่แสดงตัวอย่างและการตรวจสอบ
คลิก **(Show Preview 👁️)** ในแถบหัวเรื่องเพื่อเข้าสู่โหมดแบ่งหน้าจอ:

- **การแสดงผลแบบเรียลไทม์**: การแก้ไขทางด้านซ้ายจะสะท้อนให้เห็นทันทีในรูปแบบข้อกำหนดที่เป็นทางการทางด้านขวา
- **ป้ายกำกับความถูกต้อง**: ระบบจะทำเครื่องหมายรายการที่ตรวจสอบแล้วด้วยไอคอน **(Check ✅)**
- **การสร้าง PDF**: เมื่อตรวจสอบความถูกต้องแล้ว คุณสามารถดาวน์โหลดแบบฟอร์มข้อกำหนดการรับมอบ PDF อย่างเป็นทางการได้โดยตรงจากพื้นที่แสดงตัวอย่าง

## 5. เครื่องมือตรวจสอบประวัติบนคลาวด์ (ขั้นตอนการดำเนินงาน)
**(Cloud History ☁️)** ที่อยู่ในแถบด้านข้างหรือแถบเครื่องมือด้านล่างคือหัวใจสำคัญของความรู้ในระบบ:

- **การติดตามสถานะ**:
    - **(Pending 🔵)**: ไฟล์อยู่ในคิวรอการวิเคราะห์โดย AI
    - **(Completed 🟢)**: การแยกวิเคราะห์เสร็จสิ้น สร้างดัชนีทางเทคนิคแล้ว
    - **(Failed 🔴)**: การแยกวิเคราะห์ล้มเหลว มักเกิดจากการเข้ารหัสไฟล์หรือเนื้อหาไม่สมบูรณ์
- **การขุดข้อมูลซ้ำขั้นสุด (Force Reparse)**: สำหรับไฟล์ที่ล้มเหลวหรือมีข้อมูลน้อย ให้เลือกไฟล์เหล่านั้นแล้วคลิก **(Repeat 🔄)** เพื่อบังคับให้ AI ทำการสแกนอย่างละเอียด
- **การอ้างอิงหลายภาษา**: ในโหมดที่ไม่ใช่จีนตัวเต็ม เครื่องมือตรวจสอบจะแปลชื่อไฟล์แบบไดนามิก คลิกไฟล์เพื่อแปลเนื้อหาทางเทคนิคเป็นภาษาปัจจุบันและนำเข้าสู่แบบฟอร์ม
